# ADB Studio

ADB Studio 是一个基于 Python 和 Tkinter 的 Windows 图形化 ADB 工具，用于简化 Android 设备的常用操作。

## 主要功能

- **设备管理**：检测并选择 USB 或网络连接的 Android 设备，支持通过 IP 地址连接，支持重启 ADB 服务
- **安装 APK/XAPK**：从 `apk` 目录选择 `.apk` 或 split APK 型 `.xapk` 并安装到当前设备
- **卸载应用**：下拉框显示当前设备已安装且在配置中的应用，**附带可读名称**，精确卸载
- **导出应用**：选择当前设备上的第三方应用（**显示应用名称**），将主 APK 和所有 split APK 完整导出到指定文件夹
- **推送文件**：选择本地文件推送到设备指定路径
- **拉取文件**：从设备拉取文件到本机 `ReadByPhone` 目录
- **ADB 终端**：内置命令输入框，支持执行任意 ADB 命令，自动附加当前设备序列号；左侧提供常用命令速查列表，点击即填入
- **配置管理**：通过界面管理卸载包名清单和前缀筛选规则，无需手动编辑配置文件

## 环境要求

- Windows
- Python 3，并支持 Tkinter
- Android SDK Platform Tools，确保 `adb` 命令已加入系统 `PATH`
- Android 设备已开启 USB 调试并授权当前电脑

## 快速开始

1. 克隆或下载本项目。
2. 将需要安装的 `.apk` 或 `.xapk` 文件放入项目根目录下的 `apk` 文件夹。
3. 使用 USB 连接 Android 设备，并确认设备已授权 USB 调试。
4. 双击 `start.bat` 启动工具。
5. 从顶部设备下拉框选择已检测到的设备。
6. 在导航栏选择功能标签，所有操作自动作用于当前选中设备。

## 通过 IP 连接设备

在顶部工具栏 IP 输入框中填写设备地址，点击「连接」。

- 只填写 IP 时，例如 `192.168.1.100`，默认使用端口 `5555`
- 也可以填写完整地址，例如 `192.168.1.100:4444`
- 连接成功后自动刷新设备列表并选中新设备

> `adb connect` 不能自动为设备开启无线调试。设备必须提前允许网络 ADB 连接，并与电脑处于同一网络。

## 导出应用

切换到「导出应用」标签页后：

1. 从下拉框选择当前设备上已安装的第三方应用包名。
2. 点击「选择文件夹」设置本机目标目录。
3. 点击「导出到目标文件夹」。

工具会查询该应用的全部 APK 路径，并将 `base.apk`、`split_config.*.apk` 等文件导出到 `<目标文件夹>/<应用包名>/`。系统应用不会出现在选择列表中；如果个别分包拉取失败，日志会显示成功和失败数量，并继续尝试剩余文件。

## 卸载配置

卸载下拉框的内容来源于两个配置文件，均可在「配置」标签页中通过界面管理：

| 文件 | 说明 |
|---|---|
| `uninstall_packages.txt` | 精确包名，每行一个 |
| `uninstall_package_prefixes.txt` | 包名前缀，匹配设备上所有以该前缀开头的已装应用 |

两个来源合并后，再与当前设备的已安装包取交集，下拉框中只展示设备上实际存在的包。

## ADB 终端

切换到「终端」标签，在输入框填写 `adb` 后面的参数部分，例如：

```
shell getprop ro.product.brand
shell pm list packages -3
logcat -d
```

工具自动补全 `adb -s <设备序列号>`，无需手动输入。按回车或点「执行」运行，结果输出到下方日志区。

## APK/XAPK 文件说明

`.apk` 和 `.xapk` 文件已通过 `.gitignore` 排除，不会提交到 Git 仓库。请自行将安装包放入本地 `apk` 文件夹。

当前安装功能支持普通 `.apk`，也支持 split APK 型 `.xapk`：工具会解压 `.xapk` 中的 APK 分包，并通过 `adb install-multiple -r` 安装。

`.xapk` 内如果包含 `Android/obb` 或 `Android/data` 资源，本工具暂不会自动推送这些资源；这类资源包需要后续单独支持。

## 应用名称显示

卸载和导出功能的下拉框中，应用会以 **「应用名称 (包名)」** 的格式显示，例如 `Clash Meta for Android (com.github.metacubex.clash.meta)`，而非仅显示 bundle ID。

工具会自动从设备拉取每个应用的 base.apk，通过纯 Python 解析 AndroidManifest.xml 和 resources.arsc 提取应用名称。无需安装 `aapt`，也不需要设备端 shell 管道工具。

> 名称获取需要从设备拉取 APK 文件（临时存放，用完自动删除）。包数量较多时首次加载可能稍慢，获取后的名称会在本次会话中缓存。
